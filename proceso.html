El Viaje: De Cero al Sistema Operativo

  1. Fase 1: Autodiagnóstico de Encendido (POST)

    En cuanto entra electricidad, la CPU se inicializa y su contador de programa (PC) se fija en una dirección de memoria anteriormente establecida. Esta dirección mira hacia la ROM del firmware (BIOS o UEFI). La primera orden ejecutada es el POST (Power-On Self-Test), que es un diagnóstico in-place que comprueba si el hardware importante está correcto:

    • Comprobación de la CPU (registros, flags).
    • Comprobación de la memoria RAM (detección y prueba básica de integridad).
    • Muestra la salida.
    • Comprobación de buses principales (PCI, PCIe).

    Un error en esta secuencia probocará que no arranque el ordenador, está será comunicada con pitidos (códigos POST).

  2. Fase 2: Inicialización de Firmware y Selección de Dispositivo

    Una vez pasado el POST, el firmware inicializa controladores de bajo nivel y muestra los dispositivos. Después, mira la configuración en memorias no volátiles (NVRAM, antiguamente CMOS) para determinar la secuencia de arranque (Boot Priority). La secuencia ordenada (ej. 1. SSD NVMe, 2. USB, 3. Red PXE) le dice al firmware dónde tiene buscar el código de arranque.

  3. Fase 3: Carga del Código de Arranque (La bifurcación MBR vs. UEFI)

    Este es el punto en el que se diferencia entre el arranque tradicional y el moderno:

    • Modo Legacy (BIOS/MBR): La BIOS lee el primer sector lógico (Sector 0, de 512 bytes) del Master Boot Record (MBR). La BIOS carga estos 512 bytes en una dirección de memoria física y mueve el control de la CPU. Esta acción carga la siguiente etapa del gestor de arranque.
    • Modo Moderno (UEFI/GPT): La UEFI actuando como un mini sistema operativo, tiene la capacidad de trabajar con archivos (ej. FAT32). No lee un sector fijo, sino que busca una partición en concreto llamada GUID: la EFI System Partition (ESP). Dentro de la ESP, navega por un directorio y ejecuta un archivo EFI.

  4. Fase 4: Ejecución del Gestor de Arranque (Stage 2)

    Ahora estamos en la etapad del gestor de arranque principal como por ejemplo GRUB2, o Windows Boot Manager, que ya está cargado en la RAM. Este software es un poco más complejo:

    1. Interpreta su propio archivo de configuración.
    2. Lee múltiples sistemas de archivos como ext4 o NTFS.
    3. Tiene un menú interactivo para que el usuario diferencia entre distintos sistemas operativos o kernels.
    4. Localiza la imagen del kernel y el disco RAM inicial en el sistema de archivos.

  5. Fase 5: Carga del Kernel y el Disco RAM Inicial (initrd)

    Cuando el usuario selecciona una entrada (o por defecto), el gestor de arranque carga dos elementos en la memoria RAM:

    • El Kernel: La imagen del núcleo del sistema operativo. Es responsable de la gestión de memoria, procesos y hardware.
    • El `initrd` (initial ramdisk): Es un archivo que contiene un sistema de archivos raíz temporal. Su propósito es manejar los módulos y controladores que el kernel necesita para poder montar el sistema de archivos raíz real (diferente al que ya se ha creado).

  6. Fase 6: Inicialización del Kernel y Montaje de la Raíz Real

    El control del ordenador pasa al kernel. Este se descomprime a sí mismo, se inicializa (configurando la gestión de memoria, interrupciones, etc.) y monta el "initrd" como su sistema raíz ("/") temporal para poder montar la raíz real más tarde.

    Desde esta raiz temporal, el kernel carga los módulos del "initrd", detecta el disco duro real y monta el sistema de archivos raíz permanente. Finalmente, utiliza la llamada al sistema "pivot_root" (o "switch_root") para descartar el "initrd" y adoptar el sistema de archivos permanente como su nuevo "/".

  7. Fase 7: Ejecución del Proceso "init" (Espacio de Usuario)

    Con el sistema de archivos real creado, el kernel ha completado su arranque. Su última tarea es ejecutar el primer proceso del espacio de usuario.

    En sistemas Linux modernos, este es `systemd`. Este proceso (que ya no es el kernel) se convierte en el padre de todos procesos del sistema. Lee su propia configuración e inicia, a la vez, el resto del sistema: archivos de red, montaje de sistemas de archivos secundarios y, finalmente, el entorno gráfico y el sisetma de inicio de sesión. De este modo presenta la pantalla de bienvenida al usuario.
