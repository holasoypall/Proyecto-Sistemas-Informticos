MBR: Master Boot Record

The MBR is the first physical sector of a data storage device that the computer boots from; this is done through the BIOS.
On storage devices containing boot data, partitions are allocated a maximum of 2TB due to current limitations. Their specific address on the disk sector is Cylinder 0, Head 0, Sector 1.
For the boot process, MBR consists of three parts: the master boot code, the master partition table, and the disk signature.

The tasks of the MBR can be divided into 3 parts:

  • The BIOS looks for a target device to boot the MBR from when a computer is first turned on.

  • The MBR boot code reads the partition table and identifies the “active” partition.

  • Use the boot sector of that partition to start the operating system.

MBR vs GPT

GPT (GUID Partition Table) is the modern form of partition tables, which eliminates some of the limitations of MBR.

  • MBR

    • Maximum of 4 primary partitions, maximum size of 2TB per disk.
    • Compatible with older operating systems and BIOS.
    • Suitable for hard drives up to 2TB and systems without UEFI.

  • GPT

    • It supports an unlimited number of partitions, with no 2TB limit.
    • Optimized for UEFI systems.
    • Suitable for drives larger than 2 TB and for greater capacity and flexibility.
